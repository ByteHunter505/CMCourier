## Compact Rules (Project Standards — Auto-Resolved per Sub-Agent Launch)

The orchestrator MUST inject the matching block(s) below as `## Project Standards (auto-resolved)` in any sub-agent prompt that touches the corresponding context. These rules are **derived from** the Constitution and REBIRTH; they are not additional requirements, they are pre-digested versions of those documents for sub-agent prompts.

### When touching files in `src/cmcourier/domain/`

- **ZERO external dependencies.** Pure Python stdlib only. No imports of `requests`, `pyodbc`, `pandas`, `pydantic`, or any third-party module.
- Models are dataclasses (or simple classes) with no I/O methods.
- Ports are abstract interfaces (`abc.ABC` + `@abstractmethod`).
- This layer is the foundation of Constitution Principle I — non-negotiable.

### When touching files in `src/cmcourier/services/` or `src/cmcourier/orchestrators/`

- Depend on **ports** (interfaces from `domain/ports.py`), never on concrete adapters.
- No direct I/O — no `open()`, no `requests.get()`, no `sqlite3.connect()`.
- Hard limit: function ≤ **50 lines** (Constitution Principle III, no exceptions).
- Soft tripwires: files ≤ 400 lines, classes ≤ 200 lines.
- `mypy --strict` applies.

### When touching files in `src/cmcourier/adapters/`

- Concrete implementation of a port from `domain/ports.py`.
- **Streaming over buffering** (Principle IV): `MultipartEncoder` for CMIS uploads, cursor streaming for DB reads, file iteration for CSV.
- Thread-local connections for `pyodbc` — the iSeries Access ODBC driver is **NOT thread-safe**.
- CMIS adapter: GET warmup (`?cmisselector=repositoryInfo`) before any POST; per-thread `requests.Session` with its own JSESSIONID.
- HTTP retry policy: 4xx fails fast (do not retry); 5xx exponential backoff; `ConnectionError 10053` doubles the normal delay (REBIRTH §8.7).

### When writing tests

- **Unit tests** mock ports (the abstract interfaces), never adapters directly.
- **Integration tests** use real adapters: real SQLite (file or in-memory), real CSV, Alfresco in Docker for CMIS.
- **AS400 / DB2 is NOT mocked** (Principle VI). The `CSVDataSource` adapter covers all dev/test scenarios.
- **Alfresco** validates the CMIS Browser Binding contract; it does NOT validate IBM-specific quirks (`$t!-2_BAC_*v-1`, `clbNonGroup.BAC_*`).
- **Strict TDD Mode active**: Red → Green → Refactor. No production code without a failing test pointing at it.

### When logging

- **No PII at INFO level or below** (Principle VIII): CIF, customer names, account numbers, card numbers, full file paths are masked.
- DEBUG-level PII surfaces only behind an explicit `--debug-pii` flag, only to local rotated log files, never to stdout/stderr in production.
- Route through the central masking helper (forthcoming `cli/ui/logging.py`). Do not hand-roll regex per call site.
- Structured logging via JSON formatter; one event per line.

### When committing

- **Conventional Commits** only. Types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`, `perf`, `ci`.
- **NO `Co-Authored-By` lines.** No AI attribution.
- **NO `--no-verify`** to skip hooks. If a hook fails, fix the cause and create a new commit.
- Atomic commits, one logical change each. Imperative mood. Subject ≤ 72 characters.
- Body explains *why*; diff explains *what*.

### When opening a PR

- Link to the spec under `specs/<NNN-feature-slug>/`.
- List test evidence (which tests pass, what scenarios they exercise).
- Update `CHANGELOG.md` under `[Unreleased]`.
- Title ≤ 70 characters.
- A PR adding code without a spec is rejected (Principle VII).
- A PR violating the SRP test of Principle III, or exceeding the 50-line function cap, is rejected.

### When making architectural decisions

1. Ask: "does this fit the existing constitutional principles?" If no, propose a constitutional amendment via `.specify/amendments/<NNN-slug>.md` before proceeding.
2. Ask: "is this MVP or post-MVP?" Check `docs/roadmap/POST-MVP.md` and the "What MVP Means" section. Do not silently expand MVP scope.
3. Verify before claiming. Read code and docs first. "Dejame verificar" beats "claro que sí" every time (Principle IX).

### When configuring

- Single source of truth: `config/config.yaml`, validated by Pydantic at startup.
- Credentials from environment variables only: `AS400_USERNAME`, `AS400_PASSWORD`, `CMIS_USERNAME`, `CMIS_PASSWORD`. No other env reads outside `config/env.py`.
- No hardcoded hosts, paths, ports, table names, column names, or property IDs.
- CMIS property IDs (`clbNonGroup.BAC_*`) live in `cmis.property_catalog`, never inline in code.

### When working with PII / banking data

- See above ("When logging").
- Test fixtures and sample data under `docs/samples/` and `tests/fixtures/` are synthetic or anonymized. No real CIFs, names, or account numbers in git.
- Audit trail (the tracking store) is preserved for the regulatory window the bank specifies.
